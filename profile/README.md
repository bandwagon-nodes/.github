## 迁移前先搞清楚：你现在在用哪种线路？

很多人从其他 VPS 迁过来搬瓦工，或者之前随便选了个机房，根本不知道自己目前跑的是什么线路。

简单判断方法：用 `traceroute` 看路由路径。如果沿途出现 `59.43.x.x` 这段 IP，那就是走了 CN2（AS4809）。如果看不到这类 IP，说明你在跑普通线路，速度自然一般。

更直接的方式：在 VPS 上安装 NextTrace，对三大运营商的测试 IP 分别跑一次，看路由节点到底走了哪里。

bash
# 安装 NextTrace
curl nxtrace.org/nt | bash

# 测试电信
nexttrace 219.141.136.10

# 测试联通
nexttrace 202.106.0.20

# 测试移动
nexttrace 221.179.155.161


出现 `59.43.x.x` 或者标注 `AS4809` 的节点，就是 CN2 GIA 的标志。

## 从其他 VPS 迁移到搬瓦工节点：常见问题

**Q：迁移过来 IP 被封了怎么办？**

这是很多从旧 VPS 折腾到搬瓦工的朋友会遇到的问题。搬瓦工提供付费换 IP 服务，价格约 $7-8，全自动处理，一般几小时内完成。重要的是，他们会保证不会开到已被封的 IP——所以换过来的 IP 通常比自己随机分到的更干净。

**Q：迁移到搬瓦工后速度还是不行，怎么排查？**

第一步：确认你用的是 CN2 GIA-E 套餐，不是 KVM 入门套餐。
第二步：在 KiwiVM 里确认当前机房是 DC6 或 DC9，而不是默认的普通机房。
第三步：检查 BBR 是否开启——在 VPS 上运行 `sysctl net.ipv4.tcp_congestion_control`，确认值为 `bbr`，否则手动开启。
第四步：如果运营商是联通，优先试试日本软银节点，延迟可能比洛杉矶低一半。

**Q：我是移动用户，哪个节点最适合？**

2025 年搬瓦工在 DC6/DC9 接入了移动 CMIN2 线路，移动用户的体验终于从"最差"升到了"和电信差不多"。推荐 DC6 或 DC9，晚高峰延迟和丢包都控制得很好。日本东京 CMI 线路的 The Tokyo Plan 限量版也是移动用户的好选择，但经常缺货，需要盯着补货通知。

**Q：电信用户直接选香港节点值得吗？**

香港 CN2 GIA 套餐最低 $89.99/月，是 CN2 GIA-E 套餐月均费用的约 5 倍。如果你对延迟有极致要求（30-50ms vs 150ms），或者是公司使用、成本可以报销，那确实值得。普通个人用户，CN2 GIA-E + DC6 已经够用了。
